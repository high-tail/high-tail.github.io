# Blog Post Structure Guidelines

## File & Metadata Structure

### File Naming
- **Convention**: `YYYY-MM-DD-kebab-case-title.md`
- **Location**: `_posts/` directory
- **Example**: `2025-05-15-configuring-jekyll-chirpy.md`

### Front Matter (YAML)
Every post MUST start with this block:
```yaml
---
title: [Compelling Title]
date: [YYYY-MM-DD HH:MM:SS +TZ]
description: [Short SEO summary, < 160 chars]
comments: true
mermaid: [true|false]
categories: [Main Category, Sub Category]
tags: [tag1, tag2, tag3]
---
```

## Essential Components

### Introduction
- Hook the reader with an interesting opening
- State the main topic and purpose
- Preview what the reader will learn
- Include a brief overview of the content structure

### Main Content
- Organize into logical sections
- Use subheadings to break up content
- Each section should focus on one key idea
- Include supporting examples and evidence

### Conclusion
- Summarize key points
- Reinforce the main message
- Provide a clear call-to-action
- Suggest next steps for the reader

## Content Flow

### Logical Progression
1. Start with broad context
2. Narrow focus to specific points
3. Provide detailed examples
4. Conclude with practical takeaways

### Section Organization
- Begin each section with a clear topic sentence
- Use transition phrases between sections
- Ensure smooth flow between ideas
- Maintain consistent formatting throughout

## Best Practices

### For Different Blog Types
- **How-to guides**: Use step-by-step structure
- **Listicles**: Organize content in numbered lists
- **Opinion pieces**: Present arguments with supporting evidence
- **News updates**: Follow chronological structure

### Reader Engagement
- Address reader needs directly
- Include questions to prompt thinking
- Use active voice and present tense
- Vary sentence length and structure
